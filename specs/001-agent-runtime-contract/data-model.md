# Data Model: Agent Runtime Contract (ARC)

**Date**: 2026-06-12 (updated from 2026-05-13)
**Spec**: [spec.md](spec.md)

## Entities

### AgentRuntime CR (extended)

The existing `agent.kagenti.dev/v1alpha1` AgentRuntime CRD is extended with `spec.oboMode` and `spec.mcp` fields. No dedicated `spec.arc` section; the controller derives contract content from existing config sources.

```yaml
apiVersion: agent.kagenti.dev/v1alpha1
kind: AgentRuntime
metadata:
  name: my-agent
  namespace: team1
  labels:
    kagenti.io/type: agent
spec:
  type: agent
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-agent
  oboMode: "none"                    # NEW: "none" | "token-forwarding" | "correlation-header"
  mcp:                               # NEW: MCP server references
    servers:
      - name: "weather-tool"
        url: "http://weather-tool-service.team1:8080/mcp"
      - name: "github-mcp"
        type: "catalog"
        ref: "github-mcp-server"
  skills:                            # EXISTING, mountPath now optional
    - name: "my-skill"
      image: "ghcr.io/kagenti/skills/my-skill:latest"
      # mountPath defaults to /arc/skills/my-skill/
```

**Field rules**:
- `spec.oboMode` defaults to `none`. v1alpha1 only implements `none`; other values return a `OBOModeNotSupported` status condition.
- `spec.mcp.servers[]` is optional. Each entry has a `name` and either a direct `url` or a symbolic reference (`type` + `ref`) resolved by an operator plugin.
- `spec.skills[].mountPath` is optional. Defaults to `/arc/skills/<name>/`.
- Contract version, target type, and tracing config are derived, not declared in the CR.

### ARC Contract ConfigMap

Per-agent ConfigMap generated by the controller. Contains the materialized AGENTS.md.

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: <agent>-arc-contract
  namespace: <namespace>
  labels:
    kagenti.io/type: arc-contract
    kagenti.io/agent: <agent-name>
  ownerReferences:
    - apiVersion: agent.kagenti.dev/v1alpha1
      kind: AgentRuntime
      name: <agent-name>
data:
  AGENTS.md: |
    ---
    contract_version: v1alpha1
    target_type: Deployment
    bindings:
      - identity
    arc_root: /arc
    obo_mode: none
    ---
    # Agent Runtime Contract
    ...
```

**Lifecycle**: Created when AgentRuntime CR is created. Updated when platform config changes (AgentRuntime CR update, MCP tool additions/removals, tracing config, contract version). Deleted when AgentRuntime CR is deleted (ownerReference cascade).

### ARC MCP ConfigMap

Per-agent ConfigMap with MCP server discovery data.

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: <agent>-arc-mcp
  namespace: <namespace>
  labels:
    kagenti.io/type: arc-mcp
    kagenti.io/agent: <agent-name>
  ownerReferences:
    - apiVersion: agent.kagenti.dev/v1alpha1
      kind: AgentRuntime
      name: <agent-name>
data:
  servers.json: |
    {
      "mcpServers": {
        "gateway": {
          "url": "http://mcp-gateway-broker.mcp-system:8080/mcp",
          "transport": "streamable-http"
        },
        "weather-tool": {
          "url": "http://weather-tool-service.team1:8080/mcp",
          "transport": "streamable-http"
        }
      }
    }
```

**Lifecycle**: Regenerated when AgentRuntime CR's `spec.mcp.servers[]` changes, when MCP tool Deployments with `kagenti.io/protocol=mcp` label are added/removed, or when MCP Gateway config changes. Resolver plugin failures result in partial content and an `MCPResolutionFailed` status condition.

### AGENTS.md Structure

Conforms to the [AGENTS.md open standard](https://agents.md/). YAML frontmatter with ARC extension fields, Markdown body.

**Frontmatter fields** (ARC extensions, ignored by standard tooling):

| Field | Type | Description |
|-------|------|-------------|
| `contract_version` | string | ARC version (e.g., `v1alpha1`) |
| `target_type` | string | Deployment or StatefulSet (Sandbox deferred) |
| `bindings` | list | Available ServiceBinding categories |
| `arc_root` | string | Root path for ARC content (always `/arc`) |
| `obo_mode` | string | OBO delegation mode (v1alpha1: always `none`) |

**Body sections**:

1. Platform-Provided Environment Variables (table: name, value, description)
2. Mount Paths (table: path, contents, provisioned by)
3. Auth Behavior (inbound, outbound, agent-to-agent, user-to-agent)
4. Agent Requirements (A2A card, health probes, container port)
5. MCP Server Access (reference to `/arc/mcp/servers.json`)
6. Skills (reference to `/arc/skills/`)
7. Credentials (ServiceBinding paths, no cleartext)
8. On-Behalf-Of Delegation (current mode and behavior)

### Environment Variables Injected

| Variable | Source | Injected by | Condition |
|----------|--------|-------------|-----------|
| `ARC_CONTRACT_VERSION` | Platform config | Webhook | Always |
| `ARC_AGENT_ID` | SPIFFE ID | Webhook | Always |
| `ARC_NAMESPACE` | Pod namespace | Webhook | Always |
| `ARC_MCP_CONFIG` | `/arc/mcp/servers.json` | Webhook | Always |
| `ARC_SKILLS_DIR` | `/arc/skills/` | Webhook | Always |
| `HTTP_PROXY` / `HTTPS_PROXY` | Sidecar endpoint | Webhook | Deployment/StatefulSet targets |
| `SPIFFE_ENDPOINT_SOCKET` | SPIRE CSI | Webhook | Always |
| `SERVICE_BINDING_ROOT` | `/bindings` | Webhook | Always |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTEL collector | Webhook | Tracing enabled (namespace annotation or platform config) |

### Volume Mounts Injected

| Mount Path | Source | Type | Condition |
|------------|--------|------|-----------|
| `/arc/AGENTS.md` | `<agent>-arc-contract` ConfigMap | subPath mount | Always |
| `/arc/mcp/servers.json` | `<agent>-arc-mcp` ConfigMap | subPath mount | Always |
| `/arc/skills/<name>/` | emptyDir (populated by init container) | volume mount | When skills configured |
| `$SERVICE_BINDING_ROOT/identity/` | SPIRE CSI driver | CSI volume | Always |

### Status Conditions

| Condition | Meaning |
|-----------|---------|
| `ContractReady` | AGENTS.md and servers.json ConfigMaps generated and mounted |
| `MCPResolutionFailed` | One or more symbolic MCP server references failed to resolve |
| `OBOModeNotSupported` | Requested OBO mode not yet implemented (v1alpha1) |
| `UnsupportedTargetType` | Target type not supported (Sandbox in v1alpha1) |

## Relationships

```

AgentRuntime CR (1) ──owns──▶ (1) ARC MCP ConfigMap
AgentRuntime CR (1) ──references──▶ (1) Deployment/StatefulSet
Deployment (1) ──has──▶ (N) Pods with /arc/ mounts
Controller ──watches──▶ MCP Tool Deployments (label: kagenti.io/protocol=mcp)

```

## MCP Resolver Plugin Interface

```go
// Resolver resolves a symbolic MCP server reference to a connection descriptor.
type Resolver interface {
    Name() string
    Resolve(ctx context.Context, ref ServerRef) (*ServerEntry, error)
}

// ServerRef is a symbolic MCP server reference from spec.mcp.servers[].
type ServerRef struct {
    Name string // Server name
    Type string // Reference type (e.g., "catalog", "label")
    Ref  string // Reference value (e.g., "github-mcp-server")
}

// ServerEntry is a resolved MCP server connection descriptor.
type ServerEntry struct {
    URL       string            // Resolved endpoint URL
    Transport string            // Transport type (e.g., "streamable-http")
    Auth      *AuthHint         // Optional auth hints
}
```

kagenti-teleport-setup

kosh is a CLI that builds on OpenShell by adding local sandboxing, project teleport (sync + upload to remote sandboxes), domain allowlisting, and agent deployment.

This is a self-contained setup script that installs kosh and configures OpenShell gateway access. No kubectl or cluster access required.

First Steps

After setup completes, you have the kosh CLI ready. Here's the typical workflow:

1. Create a local sandbox

# Make sure ANTHROPIC_AUTH_TOKEN is set
export ANTHROPIC_AUTH_TOKEN=<your-token>

# Pick a name for your agent workspace
export AGENT_NAME=${USER}-agent1

# Create a sandboxed environment
kosh local-sandbox create $AGENT_NAME

2. Work inside the sandbox

# Reconnect later (or from a new terminal)
kosh local-sandbox connect $AGENT_NAME

# Run Claude inside the sandbox
claude -r

# Exit the sandbox when done
exit

3. Teleport to a remote OpenShell sandbox

# Upload your project to a remote sandbox
kosh teleport -d $AGENT_NAME

# Connect to the remote sandbox
kosh sandbox connect $AGENT_NAME

# Run Claude remotely
claude -r

4. Run headless agent commands

# Execute a one-shot prompt in the remote sandbox
kosh sandbox exec -n $AGENT_NAME -- claude --dangerously-skip-permissions \
  -p 'say hi from kagenti openshell sandbox'

5. Deploy agents and tools (optional)

# Login to Kagenti backend
kosh login --kagenti-url https://kagenti-backend-kagenti-system.apps.epoc002.ete14.res.ibm.com \
  --keycloak-url https://keycloak-keycloak.apps.epoc002.ete14.res.ibm.com \
  --user dev-user --password UonNQPfcSmzPmDSP

# Deploy a tool and agent
kosh deploy tool --name weather-tool --namespace team1 \
  --image ghcr.io/kagenti/agent-examples/weather_tool:latest \
  --protocol streamable_http --port 8000 --target-port 8000

kosh deploy agent --name weather-service --namespace team1 \
  --image ghcr.io/kagenti/agent-examples/weather_service:latest \
  --protocol a2a --framework LangGraph --port 8080 --target-port 8000 \
  --authbridge --spire

# List deployed agents and tools
kosh catalog agents -n team1
kosh catalog tools -n team1

What the setup does

  1. Checks for uv, exits with install instructions if missing
  2. Validates Python 3.12+ (auto-installs if missing)
  3. Downloads kosh.py + support files to current directory
  4. Configures openshell gateway (OIDC auth mode)
  5. Installs embedded mTLS certificates (CA + client cert/key)
  6. Performs headless OIDC login via PKCE flow
  7. Persists kosh alias to ~/.zshrc / ~/.bashrc
  8. Tests with kosh sandbox list
